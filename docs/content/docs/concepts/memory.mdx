# Memory Management

Lux GPU uses a unified memory model where arrays are accessible from both CPU and GPU without explicit transfers.

## Unified Memory

### How It Works

On Apple Silicon, the CPU and GPU share the same physical memory. This means:

- No explicit data transfers between CPU and GPU
- Arrays can be created on CPU and used on GPU seamlessly
- The system handles caching and coherency automatically

```cpp
#include <mlx/mlx.h>

using namespace mlx::core;

// Created on CPU
auto a = array({1.0f, 2.0f, 3.0f, 4.0f});

// Used on GPU (no copy needed)
auto b = matmul(reshape(a, {2, 2}), reshape(a, {2, 2}));

// Result accessible on CPU
eval(b);
std::cout << b << std::endl;
```

### Memory Pooling

The allocator maintains pools of memory buffers to reduce allocation overhead:

```cpp
#include <mlx/allocator.h>

// Get current memory usage
size_t active = allocator::active_memory();
size_t peak = allocator::peak_memory();
size_t cache = allocator::cache_memory();

std::cout << "Active: " << active / 1024 / 1024 << " MB\n";
std::cout << "Peak: " << peak / 1024 / 1024 << " MB\n";
std::cout << "Cached: " << cache / 1024 / 1024 << " MB\n";
```

### Memory Limits

Set limits on memory usage to prevent OOM conditions:

```cpp
// Set memory limit (in bytes)
allocator::set_memory_limit(4ULL * 1024 * 1024 * 1024);  // 4GB

// Clear cached memory
allocator::clear_cache();
```

## Buffer Lifecycle

### Reference Counting

Arrays use reference counting for automatic memory management:

```cpp
{
    auto a = zeros({1000, 1000});  // Buffer allocated
    auto b = a;                     // Shared reference
    // a and b share the same buffer
}
// Buffer freed when last reference goes out of scope
```

### Custom Deleters

For arrays backed by external memory:

```cpp
// External buffer with custom cleanup
float* external_data = get_external_buffer();

auto arr = array(
    external_data,
    {1024, 1024},
    float32,
    [](void* ptr) {
        release_external_buffer(static_cast<float*>(ptr));
    }
);
```

## Device Placement

### Default Device

Operations run on the default device (GPU if available):

```cpp
#include <mlx/device.h>

// Check default device
auto dev = default_device();
std::cout << "Default: " << dev << std::endl;

// Set default device
set_default_device(Device::gpu);
```

### Explicit Device

Force operations to a specific device:

```cpp
auto a = array({1.0f, 2.0f, 3.0f});

// Explicit CPU computation
auto b = add(a, a, Device::cpu);

// Explicit GPU computation
auto c = add(a, a, Device::gpu);
```

## Streams

Streams provide ordered execution queues:

```cpp
#include <mlx/stream.h>

// Default stream
auto default_s = default_stream(Device::gpu);

// Create new stream
auto s = new_stream(Device::gpu);

// Operations on specific stream
auto a = zeros({1000, 1000}, float32, s);
auto b = ones({1000, 1000}, float32, s);
auto c = matmul(a, b, s);

// Synchronize stream
synchronize(s);
```

### Stream Dependencies

```cpp
auto s1 = new_stream(Device::gpu);
auto s2 = new_stream(Device::gpu);

// Operations on s1
auto a = randn({1000, 1000}, float32, s1);
auto b = matmul(a, a, s1);

// Wait for s1 before using result in s2
synchronize(s1);
auto c = add(b, b, s2);
```

## Memory Views

Views share memory without copying:

```cpp
auto a = arange(24).reshape({4, 6});

// Views share memory
auto row = a[0];              // First row (view)
auto col = a[Ellipsis, 0];    // First column (view)
auto sub = slice(a, {1, 3}, {2, 5});  // Submatrix (view)

// Check if contiguous
bool is_contig = row.flags().contiguous;
```

### Forcing Copies

When you need independent memory:

```cpp
auto a = arange(24).reshape({4, 6});
auto view = a[0];

// Force copy
auto independent = copy(view);

// Ensure contiguous layout
auto contig = contiguous(view);
```

## Tips for Efficient Memory Use

### 1. Reuse Buffers

```cpp
// Inefficient: allocates new buffer each iteration
for (int i = 0; i < 1000; ++i) {
    auto x = zeros({1000, 1000});
    process(x);
}

// Efficient: reuse buffer
auto x = zeros({1000, 1000});
for (int i = 0; i < 1000; ++i) {
    x = multiply(x, 0.0f);  // Clear in place
    process(x);
}
```

### 2. Use Appropriate Types

```cpp
// float16 uses half the memory of float32
auto a = zeros({1000, 1000}, float16);
```

### 3. Clear Cache When Done

```cpp
// After large computation
allocator::clear_cache();
```

### 4. Monitor Usage

```cpp
void debug_memory() {
    std::cout << "Active: " << allocator::active_memory() << "\n";
    std::cout << "Peak: " << allocator::peak_memory() << "\n";
    std::cout << "Cache: " << allocator::cache_memory() << "\n";
}
```
